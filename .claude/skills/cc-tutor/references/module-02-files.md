# Module 2: 파일 다루기 + 스마트 조작법

> 공식 문서: https://code.claude.com/docs/ko/tools

## WHY

{name} {title}님, 지난달 고객사 제안서 파일이 어디 있었는지 바로 기억나시나요?

기업교육 컨설턴트의 업무 폴더에는 수십 개의 제안서, 강사 프로필, 고객사 자료가 쌓여 있습니다. 필요한 파일을 찾고, 새 문서를 만들고, 기존 문서를 수정하는 것 — 이것이 Claude Code의 가장 기본적인 능력입니다.

**Before:** 폴더를 하나하나 뒤져서 파일 찾기, 빈 문서에서 처음부터 작성
**After:** "고객사 제안서 찾아줘", "지난 제안서 템플릿으로 새 문서 만들어줘"

## EXPLAIN

Claude Code는 컴퓨터의 파일을 직접 다룰 수 있습니다. 핵심 도구 3가지:

### 1. 파일 읽기 (Read)

> 📖 비유: "프로젝트 폴더에서 보고서 꺼내 읽기"

Claude에게 파일을 보여달라고 하면 내용을 읽어서 보여줍니다.

    이 파일 읽어줘: proposals/고객사-제안서.md

### 2. 파일 쓰기 (Write / Edit)

> 📖 비유: "새 문서 작성해서 저장하기"

새 파일을 만들거나 기존 파일을 수정할 수 있습니다.

    proposals 폴더에 새-고객-제안서.md 파일을 만들어줘

### 3. 파일 검색 (Glob / Grep)

> 📖 비유: "폴더에서 특정 파일이나 내용 검색하기"

파일 이름으로 찾거나, 파일 안의 내용으로 검색할 수 있습니다.

    proposals 폴더에서 고객사 관련 파일 찾아줘
    모든 제안서에서 핵심 키워드가 들어간 부분 찾아줘

### 핵심 정리

| 도구 | B2B 비유 | 예시 |
|------|---------|------|
| Read | 보고서 꺼내 읽기 | "이 제안서 내용 보여줘" |
| Write/Edit | 새 문서 작성/수정 | "제안서 초안 만들어줘" |
| Glob | 파일명으로 검색 | "proposals 폴더에 뭐가 있어?" |
| Grep | 내용으로 검색 | "고객사명이 들어간 파일 찾아줘" |

---

파일 다루기 외에도, Claude Code를 **더 똑똑하게 조작하는 방법**들이 있습니다:

### 4. 스크린샷으로 설명하기

> 📖 비유: "이거 좀 봐줘" 하면서 화면을 보여주는 것

Claude Code 대화창에 **스크린샷을 붙여넣기(Ctrl+V / Cmd+V)** 할 수 있습니다. 에러 화면, 참고할 자료, 경쟁사 커리큘럼 등을 캡처해서 바로 보여주면 말로 설명하는 것보다 훨씬 정확합니다.

    [스크린샷 붙여넣기 후] 이 교육 커리큘럼 참고해서 비슷한 구조로 제안서 만들어줘

### 5. Plan 모드 — 실행 전에 계획부터

> 📖 비유: "바로 하지 말고, 먼저 어떻게 할 건지 계획 보여줘"

큰 작업을 시킬 때, Claude가 바로 실행하면 의도와 다를 수 있습니다. **Shift+Tab을 2번** 누르면 **Plan 모드**로 전환됩니다. 이 모드에서는 Claude가 먼저 계획을 보여주고, 승인 후에만 실행합니다.

```
일반 모드:  "해줘" → 바로 실행
Plan 모드:  "해줘" → 계획 제시 → 승인 → 실행
```

### 6. "Think harder" — 더 깊이 생각하게 하기

> 📖 비유: "대충 하지 말고 진지하게 고민해봐"

복잡한 문제에서 Claude의 답이 피상적일 때, **"think harder"** 또는 **"깊이 생각해봐"**라는 키워드를 넣으면 Claude가 더 심층적으로 분석합니다.

    이 커리큘럼이 고객사 Pain Point를 제대로 해결하는지 think harder 해서 분석해줘

### 7. 대화 흐름 제어 — 중단·요약·초기화

| 조작 | 단축키/명령 | 비유 |
|------|------------|------|
| **중단** | `Escape` | "잠깐, 그만!" — Claude가 작업 중일 때 즉시 멈춤 |
| **대화 요약** | `/compact` | "지금까지 한 얘기 정리해줘" — 긴 대화를 요약해서 컨텍스트 확보 |
| **대화 초기화** | `/clear` | "새로 시작하자" — 대화 내용 완전 초기화 |

> 💡 `/compact`는 대화가 길어져서 Claude가 앞부분을 잊어버릴 때 유용합니다. 요약본을 만들어서 기억을 유지합니다.

## EXECUTE

아래 순서대로 실행해보세요:

> 💡 이번 모듈부터 **실제 받은 RFP(제안요청서)**를 사용합니다. 이후 모듈 3~5까지 이 RFP를 기반으로 실습이 이어집니다. 실제 RFP가 없다면 "기업교육 RFP 샘플을 하나 만들어줘"라고 요청해서 시작하세요.

**실습 1: RFP 문서 넣고 분석하기 (Read)**

RFP 파일을 프로젝트 폴더에 넣고 Claude에게 읽게 합니다. 방법은 3가지 중 편한 걸로:
- **파일이 있다면:** `이 RFP 읽어줘: rfp-고객사명.pdf` (PDF, Word, 이미지 모두 가능)
- **스크린샷이라면:** RFP 화면을 캡처해서 Ctrl+V / Cmd+V로 붙여넣기
- **텍스트라면:** RFP 내용을 직접 붙여넣기

    이 RFP를 분석해서 핵심 요구사항을 정리해줘:
    1. 고객사 과제
    2. 교육 목표
    3. 대상 인원/직급
    4. 일정 및 예산 조건
    5. 특별 요구사항

**실습 2: RFP 기반 제안서 초안 만들기 (Write)**

    proposals 폴더를 만들고, 방금 분석한 RFP를 기반으로 제안서 초안을 만들어줘.
    파일명은 proposals/고객사명-제안서-초안.md 로 해줘.

**실습 3: 파일 검색 (Glob)**

    proposals 폴더에 어떤 파일들이 있는지 보여줘

**실습 4: 내용 검색 (Grep)**

    proposals 폴더에서 RFP에 나온 핵심 키워드(예: 고객사명, 교육 주제 등)가 들어간 부분을 찾아줘

**실습 5: Plan 모드로 제안서 확장하기**

Shift+Tab을 2번 눌러 Plan 모드로 전환한 후 요청하세요:

    proposals 폴더의 제안서 초안을 기반으로, 아래 3개 문서를 추가로 만들어줘:
    1. 커리큘럼 상세안 (시간표 포함)
    2. 강사진 프로필
    3. 예산 견적서

Claude가 바로 실행하지 않고 **계획을 먼저 보여줍니다**. 계획을 검토한 후 승인하세요.

**실습 6: 대화 흐름 제어 체험**

아래 3가지를 순서대로 해보세요:

1. Claude가 긴 답변을 하는 중에 **Escape**를 눌러 중단해보세요
2. `/compact`를 입력해서 지금까지의 대화를 요약해보세요
3. `/clear`는 대화를 완전히 초기화합니다 — **지금은 실행하지 마세요!** (학습 진도가 사라집니다). 새 주제로 처음부터 시작할 때 사용하세요.
