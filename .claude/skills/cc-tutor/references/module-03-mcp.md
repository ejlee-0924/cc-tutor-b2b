# STEP 3-1: MCP 기초 — 연결하고 조회하기

## WHY

{name} {title}님, 제안서 하나 쓰려면 몇 개 시스템을 열어야 하나요?

세일즈맵에서 고객 정보 확인하고, 노션에서 강사 찾고, 위클리 시트에서 딜 현황 보고, 슬랙에서 히스토리 확인하고... **MCP는 이 모든 곳을 Claude가 직접 연결해서 정보를 가져오게 하는 파이프**입니다.

**Before:** 세일즈맵·구글시트·노션·슬랙 4~5개 탭을 번갈아 열며 정보 수집 (30분+)
**After:** "이 고객 관련 정보 모아줘" 한 마디로 자동 수집

## EXPLAIN

### MCP란?

> 📖 glossary 참조: MCP = "CRM 연동"

MCP(Model Context Protocol)는 Claude가 외부 시스템과 실시간으로 소통할 수 있게 해주는 파이프라인입니다.

```
{name} {title}님: "삼성전자 교육 관련 정보 모아줘"
    │
    ▼
Claude Code
    │
    ├──▶ salesmap MCP ──▶ 고객 딜 정보·파이프라인 조회
    ├──▶ Google Sheets MCP ──▶ 위클리·견적서·강사관리 시트 조회
    ├──▶ Notion MCP ──▶ 강사 DB 검색
    ├──▶ Slack MCP ──▶ 고객 채널 대화 수집
    └──▶ Google Drive MCP ──▶ 과정 폴더·제안서 파일 조회
    │
    ▼
통합 정보 제공
```

### 주요 MCP 서버와 업무 연결

| MCP 서버 | 접근하는 업무 데이터 | 활용 예시 |
|----------|---------------------|----------|
| **salesmap** | 고객 레코드, 딜 파이프라인 | "이번 달 내 담당 딜 현황 보여줘" |
| **Google Workspace** | 위클리·견적서·전략기술서·강사관리·GPT계정 시트, Drive 폴더, Calendar, Gmail | "위클리에서 집중 ACCOUNT 보여줘" |
| **Notion** | 강사 DB (477개 역량 태그) | "생성형 AI 강의 가능한 강사 찾아줘" |
| **Slack** | 팀·고객 채널 대화 | "고객 채널에서 최근 요청사항 가져와" |
| **Calendar** | 미팅·강사 일정 | "다음 주 고객 미팅 확인해줘" |

```

# STEP 3-2: MCP 실전 — 업무 문서 자동화

## WHY

{name} {title}님, 3-1에서 MCP로 정보를 **조회**하는 것을 배웠습니다.

하지만 조회만 하면 반쪽짜리입니다. 세일즈맵에서 가져온 고객 정보를 **기획문서에 바로 채우고**, 강사 정보를 **견적서에 바로 넣을 수 있다면** 어떨까요?

**Before:** 세일즈맵에서 복사 → 시트에 붙여넣기 → 수동 정리 (건당 20분)
**After:** "이 딜 기반으로 기획문서 초안 채워줘" 한 마디로 자동 작성

## EXPLAIN

### READ vs WRITE

MCP로 할 수 있는 일은 크게 두 가지입니다:

| 구분 | 설명 | 예시 |
|------|------|------|
| **READ** (조회) | 데이터를 가져와서 보여주기 | "위클리에서 이번 주 딜 현황 보여줘" |
| **WRITE** (작성) | 데이터를 문서에 채우기/생성 | "이 고객 정보로 기획문서 채워줘" |

### 10개 업무 문서 × MCP 매핑

| 업무 문서 | MCP 서버 | READ | WRITE |
|-----------|----------|------|-------|
| 전략기술서 | Google Sheets | 기존 분석 조회 | 요구사항 분석 채우기 |
| 기획문서 | Google Sheets | 체크리스트 조회 | Winning Point 채우기 |
| 견적서 | Google Sheets | 강사료 기준 조회 | 강사·시수·비용 채우기 |
| 위클리 | Google Sheets | 딜 현황·월 타겟 조회 | — (팀장이 관리) |
| 싱크업 문서 | Google Sheets | 담당자·일정 조회 | 과정 정보 채우기 |
| 전임강사 관리 | Google Sheets | 캘린더·소진율 조회 | 출강 기록 추가 |
| 강사 계약 시트 | Google Sheets | 계약 조건 조회 | — (설문 폼 연동) |
| GPT 계정 시트 | Google Sheets | 가용 계정 조회 | 사용 일정 기입 |
| 강사 DB | Notion | 강사 검색 | — (별도 관리) |
| 과정 관리 폴더 | Google Drive | 폴더 구조 조회 | 폴더 복사·생성 |

### 데이터가 흐르는 방향

```
salesmap (고객·딜 원천 데이터)
    │
    ├──▶ 전략기술서 (요구사항 분석)
    ├──▶ 기획문서 (제안 체크리스트)
    ├──▶ 견적서 (비용 산출)
    │
    └──▶ 위클리 (딜 현황 추적)

Notion 강사 DB (강사 원천 데이터)
    │
    ├──▶ 견적서 (강사료 산정)
    ├──▶ 강사 계약 시트 (계약 진행)
    └──▶ 전임강사 관리 (출강 스케줄)
```

# STEP 3-3: Context Sync — 여러 MCP 조합 파이프라인

## WHY

{name} {title}님, MCP 하나씩 쓰는 건 검색 수준입니다.

**진짜 힘은 여러 MCP를 엮을 때** 나옵니다. 세일즈맵에서 고객 정보 가져오고, 노션에서 강사 찾고, 견적서에 자동으로 넣는 것 — 이것이 Context Sync입니다.

**Before:** RFP 받으면 세일즈맵 확인 → 노션 강사 검색 → 시트에 수동 정리 → 견적 계산 (2시간+)
**After:** "이 RFP 기반으로 전략기술서 초안 + 강사 추천 + 견적 산출해줘" (10분)

## EXPLAIN

### Context Sync란?

> 📖 glossary 참조: Context Sync = "여러 소스에서 한 문서로 통합"

여러 MCP 소스에서 가져온 정보를 **하나의 업무 산출물로 통합**하는 것입니다.

| 단계 | 설명 | 예시 |
|------|------|------|
| 1. 수집 | 여러 MCP에서 관련 정보 가져오기 | salesmap + Notion + Sheets |
| 2. 정제 | 업무 목적에 맞게 필터링·정리 | 고객 요구사항에 맞는 강사만 선별 |
| 3. 통합 | 하나의 문서/산출물로 조합 | 전략기술서 초안 완성 |

### 3가지 킬러 파이프라인

{name} {title}님의 실제 업무에서 가장 효과가 큰 3가지 조합입니다:

```
파이프라인 A: RFP 분석 → 전략기술서 초안
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
salesmap (고객 정보·딜 배경)
  + Sheets 제안서 태그 시트 (유사 과정 레퍼런스)
